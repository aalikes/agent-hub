# Agent Naming Convention

Every company's agents should follow a consistent theme. Pick one theme per company.

## Themes

| Theme | Examples | Best for |
|-------|----------|----------|
| **First names** | Casey, Penny, Metro | General — human "team member" feel |
| **Mythological** | Hermes, Athena, Apollo | Regal, authoritative businesses |
| **Nature** | River, Sage, Flint | Outdoor, wellness, sustainable |
| **Urban/Street** | Brick, Grid, Atlas | City services, logistics, real estate |
| **Tech/Abstract** | Byte, Echo, Nova | SaaS, dev tools, digital agencies |
| **Historical figures** | Tesla, Curie, Turing | Innovation, research, education |
| **Company-themed** | Print, Scan, Seal | When company name suggests a domain (e.g. Metroprints → Print, LiveScan, Apostille) |
| **Color-based** | Azure, Crimson, Jade | Creative agencies, design, fashion |
| **Gemstone** | Opal, Onyx, Jasper | Luxury, jewelry, high-end retail |
| **Constellation** | Vega, Orion, Lyra | Space, data, analytics |

## Rules

1. **Pick one theme per company.** Don't mix themes — if Metroprints uses first names (Casey, Penny, Metro), all Metroprints agents use first names.

2. **Name reflects role.** A financial agent should have a name that conveys precision (e.g. Penny, Sterling). An operations agent should feel capable (e.g. Atlas, Brick). A creative agent should feel expressive (e.g. Muse, Nova).

3. **Pronunciation matters.** Names agents will be mentioned in Slack — `@Athena` is easier than `@Xochitl`. Avoid names that are hard to type or spell.

4. **1-2 syllables preferred.** Shorter names are easier to @mention quickly.

5. **Check Slack availability.** The name becomes the Slack bot display name. Avoid names already taken in the workspace.
