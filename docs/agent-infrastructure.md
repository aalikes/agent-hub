# Agent Infrastructure — Multi-Agent Setup

> Standardized setup for all Slack agents.
> **For new companies:** Start with `onboarding/company-agent-onboarding-template.md` (Phase 1: Research → Phase 2: Design → Phase 3: Deploy).

## Deploy (Automated)

```bash
cd ~/Projects/agent-hub
./deploy/deploy-all.mjs companies/COMPANY.json
```

This single command deploys every agent in the config:
1. Creates Slack apps via API (if config_token provided)
2. Generates `listener.mjs` with cron, sub-agent spawning, alert routing baked in
3. Copies `listener-utils.mjs` (shared utilities)
4. Generates and loads launchd plist
5. Appends MCP config + permissions to `opencode.jsonc`
6. Generates OpenCode agent definition `.md`

**One manual step remains:** restart OpenCode to pick up MCP changes.

## Agent Directory Structure

Each agent:
```
~/Projects/{company}/agents/{name}/
  listener.mjs          # Socket Mode listener (cron + spawning + alerts)
  listener-utils.mjs    # Sub-agent spawning, alert routing, health checks

~/Projects/{company}/.opencode/agent/
  {name}.md             # OpenCode agent definition

~/Library/LaunchAgents/
  com.{company}.{name}.listener.plist  # macOS persistence
```

## Listener Capabilities

Every agent generated by `deploy-all.mjs` ships with:

| Feature | How |
|---------|-----|
| **Cron engine** | Parses cron expressions, schedules jobs, auto-reschedules |
| **Sub-agent spawning** | Detects "spawn agent" in @mentions, delegates via `listener-utils.mjs` |
| **Alert routing** | P0-P3 severity → correct channel with emoji formatting |
| **Context folding** | Summarizes old messages at ~15K tokens |
| **Thread tracking** | 30-min active thread window, no @mention needed in active threads |
| **Deduplication** | Prevents double-processing of app_mention + message events |
| **Health checks** | Self-diagnostics: Slack connection, LLM API, agent status |
| **Obsidian knowledge** | Loads markdown files from vault into system prompt |
| **Notion integration** | Search and query databases if API key configured |
| **Slash commands** | Registered via Slack manifest, handled by LLM |

## Slack App Scopes

Required (included by default):
- `channels:read`, `channels:history`, `groups:read`, `groups:history`
- `users:read`, `chat:write`, `app_mentions:read`
- `im:history`, `im:read`, `im:write`
- `reactions:read`, `reactions:write`
- `files:read`, `commands`, `usergroups:read`

Optional (add in company config if needed):
- `channels:manage`, `channels:join`, `groups:write`
- `files:write`, `assistant:write`

## Management Commands

```bash
# Start
launchctl load ~/Library/LaunchAgents/com.{company}.{agent}.listener.plist

# Stop
launchctl unload ~/Library/LaunchAgents/com.{company}.{agent}.listener.plist

# Status
launchctl list | grep {agent}

# Logs
tail -f ~/Library/Logs/com.{company}.{agent}.listener.log
tail -f ~/Library/Logs/com.{company}.{agent}.listener.error.log
```

## Scope Audit

```bash
rg -oP 'slack\.com/api/\K\w+\.\w+' ~/Library/Logs/com.{company}.{agent}.listener.log \
  | sort | uniq -c | sort -rn
```
